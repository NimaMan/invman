Motivation

For a century, inventory control advanced by proving the optimal policy has a simple, computable form.

Start with the lineage. Each landmark below reduced a hard sequential decision to a closed-form control rule — an order quantity, an order-up-to level, a pair of thresholds — that could be written down and computed.

One simple, computable rule per landmark. A lot size, an order-up-to level, a pair of thresholds — each result turned a hard dynamic program into a control law that could be written down and computed.

Motivation

Lead times and network structure make the inventory decision state-dependent and high-dimensional.

Look at the state up close. Each period asks for one operational action, yet the state must remember everything already in transit. Two structures enlarge it together: a lead-time pipeline that carries one bucket per period of delay, and a one-warehouse to K-retailer network that adds a coordinate per location.

One action per period, over a state that scales with both delay and breadth. The lead-time pipeline grows the state with the lead time L; the network grows it with the retailer count K, so raw state and action spaces enlarge along two axes at once.

Method

CMA-ES minimizes realized operating cost directly on rollouts of a compact policy.

One gradient-free loop, held fixed across all ten problems. The search distribution proposes compact policy parameters; the simulator returns realized cost; CMA-ES updates the search. It adapts the full covariance of that distribution and is rank-based, needing no gradients, value function, or discount factor on a non-smooth, noisy simulator — the appropriate optimizer for tens to a few hundred parameters. The only per-problem design is the decoder shown next.

The loop ranks proposals by realized cost and shifts the search toward the cheaper ones. Only the decoder is designed per problem.

CMA-ES, Hansen & Ostermeier, 2001 · ES as a scalable RL alternative, Salimans et al., 2017 · population 64
Policy parameterization

The decoder sets which actions a policy can reach.

One learner, one backbone, one policy class are held fixed; only the bracketed decoder block changes the reachable action set. Structured action classes are long-standing — base-stock and (s, S) (Scarf, 1960), echelon levels (Clark & Scarf, 1960), capped dual-index (Sun & Van Mieghem, 2019), action-space shaping (Kanervisto et al., 2020), action representations (Chandak et al., 2019), residual policies (Johannink et al., 2019), and structure-informed DRL (Maggiar et al., 2025) — here re-expressed as compact, policy-owned decoders tuned by one gradient-free recipe.

Same learner, same backbone, same policy class. The decoder block alone fixes the reachable action set; the case studies instantiate it.

Synthesis

The recipe is fixed; only the decoder changes shape across problems.

One gradient-free loop — state → normalize → backbone → action — is held fixed across every problem. Only the single bracketed decoder slot takes a problem-specific shape. The same slot carries across all ten families; the three drawn below are illustrative, and the two hardest geometries — the heterogeneous multi-retailer and the mixed-assembly head — were found by a search over decoder classes, not hand-designed.

One loop, three decoder shapes. The compact policy searches in a parameterization aligned with each problem's control structure; the four fixed nodes are shared, and only the bracketed decoder takes a problem-specific form.
