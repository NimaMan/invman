## Honest status: literature_verified = false

No test in this family reproduces a number printed in the Dehaybe et al. (2024)
EJOR article. The repo rule treats reference-implementation matches and internal
self-consistency checks as NOT literature verification, and that is exactly what
these tests are.

What the verifier asserts:

1. reference-set shape
   - 8 forecast definitions are present
   - 8 benchmark instances are present
   - the primary reference instance is `dehaybe2024_lostsales_lt2_b5_k10_constant_10`
2. observation and mechanics checks (INTERNAL self-consistency, not paper numbers)
   - the policy-state layout matches the Section 4.1 formulation
   - `worked_example_transition_is_internally_consistent`: `step_state` recomputes
     the Section-4 illustrative transition reward `-130` from our own cost
     arithmetic. We do NOT claim the article prints `-130`; the EJOR full text was
     not accessible.
3. simple baseline checks (REFERENCE-IMPLEMENTATION match)
   - the literature (s,S) formula gives `(s, S) = (33.351246609652, 47.49338223338295)`
     on the primary constant-10 window
   - `simple_s_s_matches_author_testbed_csv_row_within_tolerance`: the simulated
     simple baseline reproduces the author's public testbed CSV row
     (`scarf_testbed_simple_lostsales.csv`) within the stored tolerances
4. rolling-DP checks (REFERENCE-IMPLEMENTATION match)
   - the first-period rolling-DP levels are generated by the Rust solver and
     regression-checked in the verifier
   - `rolling_dp_matches_author_testbed_csv_row_within_tolerance`: the rolling-DP
     simulation reproduces the author's public testbed CSV row
     (`scarf_testbed_DP_lostsales.csv`) within the stored tolerances
5. drift guard
   - `honest_status_flags_are_false`: asserts every `literature_verified` flag is
     `false` and every `verification_source` is unchanged, so the status cannot
     silently drift to an overclaim.

Verification semantics:

- the worked transition is an internal `step_state` mechanics check, not a
  literature number
- the benchmark-row assertions use the author's public companion-code testbed
  CSVs (reference implementation) and the tolerances in
  `VERIFICATION_PROBLEM_INSTANCE`; they are NOT article-printed table values
