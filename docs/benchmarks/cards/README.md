## Difficulty rubric

Difficulty folds three axes: (a) state/action dimensionality, (b) exact-solver availability (an exact VI/DP true optimum makes a problem easier to benchmark), and (c) comparator type (`true_optimum_match_only` easiest → `heuristic_to_beat` ~ `bound_gap` → `self_consistent` hardest). The per-problem `difficulty` + one-line `difficulty_rationale` live in the manifest; the full rubric + the three verification tiers are documented in `../README.md`.
